Contribua

Software livre só existe porque pessoas decidem construir em conjunto. Há várias formas de ajudar — com código, com ideias ou simplesmente acompanhando.

Como ajudar

Formas de contribuir

Não é preciso ser desenvolvedor para contribuir. Cada uma dessas formas avança o projeto de maneira real.

💻
Código

Implemente features, corrija bugs, melhore a cobertura de testes ou refatore algo que está difícil de manter. O projeto usa Python, Django e LangGraph — mas contribuições em documentação técnica e arquitetura também contam.

Ver repositório →
📝
Documentação

Escreva tutoriais, melhore os docs existentes, traduza conteúdo ou explique conceitos complexos de forma mais simples. Boa documentação é o que torna um projeto acessível a quem quer contribuir.

Ver documentação →
🧪
Teste e feedback

Use o sistema, tente quebrar coisas, relate o que não funciona como esperado. Feedback de quem não está no projeto é mais valioso do que parece — você vê o que quem está dentro não vê.

Reportar algo →
💡
Ideias e discussões

Tem uma ideia de caso de uso, uma persona que não está sendo atendida ou uma direção arquitetural para discutir? Abra uma issue ou use as Discussions do GitHub. Boas ideias mudam a trajetória do projeto.

Abrir discussão →
✍️
Diário de bordo

Se você está contribuindo com o projeto, escreva sobre o que está fazendo no diário. Registros diretos dos integrantes são o que mantém o projeto vivo e transparente para quem está de fora.

Ver o diário →
📢
Espalhar a palavra

Compartilhe o projeto com quem possa se beneficiar ou queira contribuir. Professores, servidores públicos, jornalistas, pesquisadores, desenvolvedores. O projeto cresce quando as pessoas certas ficam sabendo que ele existe.
